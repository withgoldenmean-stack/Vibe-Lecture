# UI 설계서

## 1. 문서 목적
본 문서는 AX/DX 바이브코딩 20시간 과정 웹 서비스의 화면 구조, 메뉴 구성, 반응형 UI를 정의한다.

## 2. 전체 화면 구조
```text
┌────────────────────────────────────────────┐
│ Top Menu                                    │
│ 서비스명 | 단원/강의 메뉴 | 검색(2차)       │
└────────────────────────────────────────────┘

┌───────────────────┬────────────────────────┐
│ Left Menu          │ Body                   │
│ 세부목차            │ 상세교육내용             │
│ 산출물 요약          │ 실습 과제                │
└───────────────────┴────────────────────────┘
```

## 3. Top Menu 설계
| 영역 | 내용 |
|---|---|
| 좌측 | 서비스명 |
| 중앙 | 단원 또는 교시 메뉴 |
| 우측 | 검색 버튼, 전체목차 버튼, 2차 기능 |

서비스명: `AX/DX Vibe Coding Course`

MVP에서는 20개 강의를 모두 가로로 나열하지 말고 단원별 탭 또는 드롭다운 구조를 권장한다.

## 4. Left Menu 설계
### 표시 내용
- 현재 선택 강의명
- 교시 번호
- 강의 산출물
- 세부목차 목록

### 선택 상태
현재 선택된 세부목차는 배경색 변경, 왼쪽 포인트 바, 글자 굵게 처리로 강조한다.

## 5. Body 영역 설계
| 순서 | 요소 |
|---|---|
| 1 | 단원명 |
| 2 | 교시 번호와 강의명 |
| 3 | 세부목차 제목 |
| 4 | 상세 교육내용 |
| 5 | 실습 과제 |
| 6 | 산출물 |
| 7 | 이전/다음 버튼 |

## 6. PC 화면 설계
| 영역 | 너비 |
|---|---|
| Left Menu | 280px |
| Body | 나머지 전체 |
| Top Menu | 100% |

## 7. 모바일 화면 설계
```text

↓
현재 강의 선택 드롭다운
↓
세부목차 선택 드롭다운
↓
Body 콘텐츠
```

## 9. Typography & Design System
본 서비스는 모든 화면에서 동일한 Typography System, Font Scale, Design Token을 사용한다. 페이지별 또는 컴포넌트별 임의 폰트와 임의 크기 사용을 금지한다.

### 9.1 기본 폰트
| 언어/용도 | Font Family |
|---|---|
| 한글 | Pretendard, Noto Sans KR, sans-serif |
| 영문 | Inter, sans-serif |
| 실제 CSS 우선순위 | Pretendard, Inter, Noto Sans KR, Apple SD Gothic Neo, system-ui, sans-serif |

### 9.2 Font Weight 규칙
| 용도 | Weight | Tailwind |
|---|---:|---|
| Page Title | 700 | `font-bold` |
| Section Title | 600 | `font-semibold` |
| Card Title | 600 | `font-semibold` |
| Body | 400 | `font-normal` |
| Caption/설명 문구 | 400 | `font-normal` |
| Button | 500 | `font-medium` |

### 9.3 Font Size System
Tailwind 기본 토큰만 사용한다. `text-[17px]`, `text-[19px]` 같은 임의 크기는 사용하지 않는다.

| Token | Size | 주요 사용처 |
|---|---:|---|
| `text-sm` | 14px | 보조 문구, 작은 라벨 |
| `text-base` | 16px | 일반 본문, 입력, 버튼 |
| `text-lg` | 18px | 카드 제목 |
| `text-xl` | 20px | 섹션 제목 |
| `text-2xl` | 24px | 페이지/본문 제목 |
| `text-3xl` | 30px | 대형 제목이 필요한 경우 |
| `text-4xl` | 36px | 랜딩 Hero 전용 |

모바일에서는 14px 미만 텍스트를 사용하지 않는다. 따라서 모바일에 노출되는 UI에는 `text-xs`를 사용하지 않는다.

### 9.4 Line Height
| 용도 | 기준 | Tailwind 권장 |
|---|---:|---|
| Title | 1.2 | `leading-tight`, `leading-snug` |
| Body | 1.6 | `leading-relaxed` |
| Caption | 1.5 | `leading-normal` |

### 9.5 색상 토큰
| Token | Color | 용도 |
|---|---|---|
| `point` | #2563EB | Primary, 선택 상태, 주요 CTA |
| `success` | #16A34A | 성공, 산출물, 완료 상태 |
| `warning` | #F59E0B | 주의, 실습 활동 |
| `danger` | #DC2626 | 오류, 경고 메시지 |
| `ink` | #111827 | 주 텍스트 |
| `muted` | #6B7280 | 보조 텍스트 |
| `line` | #E5E7EB | 경계선 |
| `canvas` | #F8FAFC | 페이지 배경 |

### 9.6 컴포넌트 적용 규칙
| 컴포넌트/요소 | Typography |
|---|---|
| TopNavigation 서비스명 | `text-lg font-semibold` |
| Body 세부목차 제목 | `text-2xl font-bold` |
| Body 섹션 제목 | `text-xl font-semibold` |
| 카드 제목 | `text-lg font-semibold` |
| 본문 | `text-base font-normal leading-relaxed` |
| 설명 문구 | `text-sm font-normal text-muted` |
| Button | `text-base font-medium` |
| Input/Select | `text-base font-normal` |

### 9.7 개발 금지 사항
- 페이지마다 다른 폰트 사용 금지
- Tailwind arbitrary font size 사용 금지
- 모바일 노출 UI에서 12px 이하 텍스트 사용 금지
- 버튼마다 다른 font weight 사용 금지

## 11. UI 완료 기준
- 사용자가 현재 어느 강의를 보고 있는지 알 수 있다.
- 사용자가 현재 어느 세부목차를 보고 있는지 알 수 있다.
- 20개 강의가 과도하게 복잡해 보이지 않는다.
- 모바일에서 메뉴 선택이 불편하지 않다.
- 본문 가독성이 확보된다.
- 어느 화면으로 이동하더라도 동일한 Typography System과 Design Token이 유지된다.
