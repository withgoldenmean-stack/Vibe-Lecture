# 바이브코딩용 개발지시서

## 1. 개발 요청
당신은 숙련된 프론트엔드 개발자입니다.

Next.js, TypeScript, Tailwind CSS를 사용하여 「바이브코딩 활용한 AX/DX 자동화 기본과 실무 과정」의 20시간 강의 목차와 상세 교육내용을 제공하는 반응형 웹 서비스를 개발해 주세요.

## 2. 서비스 목적
비개발자 AX/DX 관심자가 20시간 강의 과정의 전체 흐름, 교시별 세부목차, 상세 교육내용, 산출물을 웹에서 쉽게 탐색할 수 있도록 합니다.

## 3. 핵심 화면 구조
화면은 다음 구조로 만듭니다.

```text
상단 Top Menu
하단 좌측 Left Menu
하단 우측 Body
```

### Top Menu
- 서비스명 표시
- 단원 탭 표시
- 강의 선택 메뉴 표시

### Left Menu
- 선택된 강의명 표시
- 교시 번호 표시
- 산출물 표시
- 세부목차 목록 표시

### Body
- 선택한 세부목차의 상세교육내용 표시
- 실습 활동 표시
- 산출물 표시
- 이전/다음 이동 버튼 표시

## 4. 반드시 구현할 기능
- 20개 강의 데이터 표시
- Top Menu에서 강의 또는 단원 선택
- Left Menu에서 세부목차 선택
- Body에 상세교육내용 출력
- 현재 선택된 강의와 세부목차 강조
- 반응형 웹 레이아웃
- 모바일에서는 1열 구조로 전환
- 데이터가 없을 경우 빈 상태 메시지 표시
- 강의별 산출물 표시

## 5. MVP 제외 기능
회원가입, 로그인, 결제, 관리자 CMS, 동영상 플레이어, 댓글, 질문게시판, 수료증, AI 챗봇은 만들지 마세요.

## 9. 상태 관리
React useState를 사용해 다음 상태를 관리합니다.

```ts
selectedUnit
selectedLessonId
selectedSectionId
```

단원 선택 시 해당 단원의 첫 번째 강의를 선택합니다.
강의 선택 시 해당 강의의 첫 번째 세부목차를 선택합니다.
세부목차 선택 시 Body 내용이 변경됩니다.

## 10. 디자인 요구사항
- 전체적으로 교육용 웹 서비스처럼 깔끔하고 신뢰감 있게 구성합니다.
- 배경은 밝은 회색, 콘텐츠 카드는 흰색으로 구성합니다.
- 현재 선택된 메뉴는 파란색 계열로 강조합니다.
- 본문은 가독성을 위해 최대 폭을 제한합니다.
- 표, 목록, 카드 스타일을 적극 활용합니다.
- 모바일에서도 읽기 쉬운 글자 크기와 여백을 사용합니다.

## 11. 반응형 요구사항

```

Mobile:

```

## 13. 완료 기준
- `npm run dev`로 실행 가능해야 합니다.
- 20개 강의가 모두 표시되어야 합니다.
- Top Menu 클릭 시 강의 목록이 바뀌어야 합니다.
- Left Menu 클릭 시 Body 내용이 바뀌어야 합니다.
- 현재 선택 상태가 시각적으로 보여야 합니다.
- 모바일에서도 정상적으로 사용할 수 있어야 합니다.
- 데이터와 화면 컴포넌트가 분리되어 있어야 합니다.
