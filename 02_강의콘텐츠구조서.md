# 강의 콘텐츠 구조서

## 1. 문서 개요

### 1.1 목적

본 문서는 20시간 AX/DX 교육과정의 강의 콘텐츠를 웹 서비스에서 일관되게 관리하고 표시하기 위한 정보 구조와 작성 기준을 정의한다.

### 1.2 적용 범위

- 20개 교시의 기본 정보와 세부 목차
- 본문, 비교표, 상세 설명, 실습, 체크리스트, 산출물
- 데스크톱 및 모바일 화면의 콘텐츠 표시
- 콘텐츠 등록·수정 시 검수 기준

### 1.3 기준 데이터

현재 웹 서비스의 실제 강의 데이터는 `src/data/lessons.ts`에서 관리하며, 타입 정의는 `src/types/lesson.ts`를 기준으로 한다.

> 이 문서는 콘텐츠 설계 기준을 설명하는 문서이다. MD 파일을 수정하는 것만으로 웹 화면의 강의 데이터가 자동 변경되지는 않는다.

## 2. 전체 과정

### 2.1 과정명

**바이브코딩을 활용한 AX/DX 기본과 실무 과정**

### 2.2 교육 대상

- AX/DX를 업무에 적용하려는 비개발자
- AI와 자동화를 활용해 업무 서비스를 기획하려는 실무자
- 바이브코딩으로 MVP 제작을 준비하는 강사·컨설턴트·기획자

### 2.3 과정 목표

수강생은 자신의 업무에서 AX/DX 과제를 발굴하고, 서비스 기획·개발·운영에 필요한 핵심 문서를 단계적으로 작성하여 바이브코딩 도구에 전달할 수 있는 수준의 개발지시서를 완성한다.

## 5. 교시별 기본 구조

각 교시는 다음 정보를 가진다.

| 필드 | 타입 | 필수 | 설명 |
|---|---|:---:|---|
| `lessonId` | string | O | 강의 고유 ID. `L001` 형식 |
| `lessonNo` | number | O | 교시 번호. 1~20 |
| `unit` | UnitName | O | 소속 단원 |
| `title` | string | O | 강의명 |
| `duration` | string | O | 강의 시간 |
| `target` | string | O | 주요 교육 대상 |
| `output` | string | O | 교시에서 완성할 산출물 |
| `summary` | string | O | 강의 핵심 내용을 요약한 문장 |
| `sections` | LessonSection[] | O | 세부 목차 목록 |

```ts
type Lesson = {
  lessonId: string;
  lessonNo: number;
  unit: UnitName;
  title: string;
  duration: string;
  target: string;
  output: string;
  summary: string;
  sections: LessonSection[];
};
```

## 6. 세부 목차 구조

모든 교시는 학습 흐름을 통일하기 위해 다음 5개 세부 목차를 기본으로 사용한다.

| 순서 | 세부 목차 | 역할 |
|---:|---|---|
| 1 | 학습 목표 | 학습 내용, 도달 목표, 산출물 확인 |
| 2 | 핵심 개념 | 핵심 용어, 원리, 비교 기준 설명 |
| 3 | 실무 예시 | 실제 업무 장면과 적용 사례 제시 |
| 4 | 실습 활동 | 수강생이 직접 작성하거나 판단하는 과제 |
| 5 | 정리 및 산출물 | 핵심 내용 정리, 결과물 완성, 다음 교시 연결 |

### 6.1 세부 목차 필드

| 필드 | 타입 | 필수 | 설명 |
|---|---|:---:|---|
| `sectionId` | string | O | 세부 목차 고유 ID. `S001-01` 형식 |
| `title` | string | O | 세부 목차명 |
| `content` | string | O | 핵심 본문 |
| `activity` | string | X | 수강생 실습 또는 적용 질문 |
| `checklist` | string[] | X | 학습 완료 여부를 확인하는 항목 |
| `comparisonTable` | object | X | 핵심 비교 정보를 보여 주는 표 |
| `detailBlocks` | LessonDetailBlock[] | X | 제목·문단·인용·목록·흐름·표로 구성한 상세 콘텐츠 |

```ts
type LessonSection = {
  sectionId: string;
  title: string;
  content: string;
  activity?: string;
  checklist?: string[];
  comparisonTable?: {
    title: string;
    columns: string[];
    rows: string[][];
  };
  detailBlocks?: LessonDetailBlock[];
};
```

## 7. 상세 콘텐츠 블록

긴 강의 내용을 하나의 문자열에 몰아넣지 않고 의미 단위로 나누어 표시한다.

| 블록 유형 | 용도 | 주요 필드 |
|---|---|---|
| `heading` | 내용 안의 소제목 | `text`, `level` |
| `paragraph` | 설명 문단 | `text` |
| `quote` | 정의, 핵심 메시지, 강조 문장 | `text` |
| `list` | 핵심 항목 또는 설명 목록 | `items` |
| `flow` | 업무·실습 절차 | `title`, `steps` |
| `table` | 비교, 기준, 사례 정리 | `title`, `columns`, `rows` |

작성 원칙:

- 한 블록에는 하나의 메시지만 담는다.
- 표의 모든 행은 열 개수와 같아야 한다.
- 절차는 실제 수행 순서대로 작성한다.
- 인용 블록은 핵심 정의나 기억해야 할 문장에만 사용한다.
- 모바일 가독성을 위해 표의 열 수는 가급적 4개 이하로 구성한다.

```ts

  sections: [

      activity:

      },
      detailBlocks: [
        {
          type: "quote",
          text: "DX가 업무를 시스템으로 바꾸는 것이라면, AX는 그 시스템이 더 똑똑하게 일하도록 만드는 것이다."
        },
        {
          type: "flow",
```

## 9. 콘텐츠 표시 원칙

- 상단 영역에는 과정명, 현재 교시, 강의명, 전체 강의 수를 표시한다.
- 단원 탭에는 현재 선택된 단원을 시각적으로 구분한다.
- 강의 선택 영역에는 `lessonNo`, `title`, `output`을 표시한다.
- 왼쪽 메뉴에는 선택한 강의의 요약, 산출물, `sections.title`을 표시한다.
- 본문에는 `content`, `comparisonTable`, `detailBlocks`, `activity`, `checklist`, `output` 순으로 표시한다.
- 모바일에서는 단원·강의·세부 목차를 선택형 UI로 제공하고 본문을 1열로 표시한다.
- 등록된 데이터가 없을 때는 상황에 맞는 빈 상태 메시지를 표시한다.

권장 빈 상태 메시지:

- 강의 없음: `등록된 강의가 없습니다.`
- 세부 목차 없음: `등록된 세부목차가 없습니다.`
- 상세 내용 없음: `등록된 상세내용이 없습니다.`

## 10. 콘텐츠 작성 원칙

### 10.1 문장

- 비개발자가 이해할 수 있는 평이한 표현을 사용한다.
- 한 문장에는 가급적 하나의 의미만 담는다.
- 추상적인 설명 뒤에는 반드시 업무 예시를 제시한다.
- 명사형 표현을 반복하기보다 실제 행동이 드러나는 동사를 사용한다.

### 10.2 실습

- 60분 안에 초안을 완성할 수 있는 범위로 설계한다.
- 실습 지시문에는 대상, 행동, 결과물 형식을 포함한다.
- 다음 교시에서 이어 쓸 수 있도록 산출물 간 연결 관계를 명시한다.

### 10.3 체크리스트

- `이해했다`보다 `설명할 수 있다`, `작성했다`, `구분할 수 있다`처럼 확인 가능한 문장으로 작성한다.
- 교시당 2~5개 항목을 권장한다.

## 11. ID 및 명명 규칙

| 항목 | 규칙 | 예시 |
|---|---|---|
| 강의 ID | `L` + 세 자리 교시 번호 | `L001`, `L020` |
| 세부 목차 ID | `S` + 세 자리 교시 번호 + `-` + 두 자리 순번 | `S001-01`, `S020-05` |
| 단원명 | 정의된 `UnitName`만 사용 | `AX/DX 기획` |
| 강의명 | 행동 또는 학습 주제가 드러나게 작성 | `AX/DX 과제 발굴` |
| 산출물명 | 수강생이 제출할 결과물 이름으로 작성 | `AX/DX 후보 업무표` |

허용 단원명:

```text
입문
AX/DX 기획
개발문서 설계
AI·API·리스크
운영·출시
```

## 12. 콘텐츠 검수 기준

- 1~20교시가 빠짐없이 등록되어 있는가?
- 강의 ID와 세부 목차 ID가 규칙에 맞고 중복되지 않는가?
- 모든 강의에 5개 기본 세부 목차가 있는가?
- 모든 강의에 제목, 요약, 대상, 시간, 산출물이 있는가?
- 모든 세부 목차에 제목과 본문이 있는가?
- 표의 열 수와 각 행의 셀 수가 일치하는가?
- 실습 지시가 구체적이며 해당 교시 산출물과 연결되는가?
- 체크리스트가 실제 확인 가능한 행동으로 작성되었는가?
- 같은 용어와 단원명이 문서와 코드에서 일관되게 사용되는가?
- 데스크톱과 모바일에서 긴 문장과 표를 읽을 수 있는가?

검증 명령:

```bash
npm run validate:data
npm run typecheck
npm run build
```
