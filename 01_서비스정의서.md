# 서비스정의서

## 1. 서비스명
AX/DX Vibe Coding Course Navigator

국문명: 바이브코딩 AX/DX 자동화 교육 내비게이터

## 2. 한 줄 정의
AX/DX와 바이브코딩에 관심 있는 비개발자가 20시간 강의 과정의 전체 목차, 단원별 세부목차, 상세 교육내용을 반응형 웹 화면에서 직관적으로 탐색할 수 있도록 돕는 교육 콘텐츠 웹 서비스이다.

## 3. 서비스 배경
「바이브코딩 활용한 AX/DX 자동화 기본과 실무 과정」은 AX/DX 개념 이해부터 AX/DX 과제 발굴, 서비스 정의서, 기능명세서, UI 설계서, 데이터 구조, AI 기능 설계, API 연동, 보안·개인정보·운영 리스크, 개발지시서 작성까지 이어지는 20시간 과정이다.

기존 강의안이 문서 파일 중심으로 제공되면 학습자는 원하는 교시와 세부목차를 찾기 어렵고, 강사는 매번 목차와 자료 위치를 반복 안내해야 한다.

## 4. 목표 고객
| 구분 | 설명 |
|---|---|
| 1차 사용자 | AX/DX와 바이브코딩에 관심 있는 비개발자 |
| 교육생 | 업무 AX/DX 서비스를 만들고 싶은 실무자 |
| 강사 | 강의 흐름과 세부내용을 화면으로 안내하는 사람 |
| 교육기획자 | 20시간 과정을 커리큘럼 형태로 소개하고 관리하는 사람 |
| 기관 담당자 | 공공기관, 교육기관, 지원기관에서 AX/DX 교육을 운영하는 담당자 |

## 5. 고객 문제
| 문제 | 설명 |
|---|---|
| 전체 흐름 파악 어려움 | 20시간 강의가 길고 교시 간 연결 구조가 복잡하다. |
| 세부내용 탐색 불편 | 문서 파일을 열어 원하는 목차를 찾아야 한다. |
| 모바일 가독성 부족 | 긴 강의안 문서는 모바일에서 보기 어렵다. |
| 교육 운영 비효율 | 강사가 목차와 자료 위치를 반복 설명해야 한다. |
| 향후 확장 어려움 | 콘텐츠가 구조화되지 않으면 검색, 진도관리, CMS 확장이 어렵다. |

## 6. 해결 가치
| 가치 | 설명 |
|---|---|
| 빠른 탐색 | Top Menu와 Left Menu로 원하는 교육내용에 빠르게 접근한다. |
| 학습 몰입 | Body 영역에는 선택한 세부목차의 상세내용만 표시한다. |
| 교육 운영 효율 | 강사와 교육생이 동일한 화면을 기준으로 수업을 진행한다. |
| 반응형 접근성 | PC, 태블릿, 모바일에서 동일한 콘텐츠를 이용한다. |
| 확장성 | 향후 검색, 로그인, 진도관리, 관리자 CMS로 확장할 수 있다. |

## 8. 사용 시나리오 요약
1. 사용자가 웹 서비스에 접속한다.
2. Top Menu에서 원하는 강의를 클릭한다.
3. Left Menu에 해당 강의의 세부목차가 표시된다.
4. 사용자가 세부목차를 클릭한다.
5. Body 영역에 상세교육내용이 출력된다.
6. 사용자는 다음 세부목차 또는 다른 강의로 이동한다.

## 9. MVP 범위
20시간 강의 데이터 등록, Top Menu 구현, Left Menu 구현, Body 콘텐츠 출력, 현재 선택 메뉴 강조, 반응형 웹 레이아웃, 빈 상태 안내를 포함한다.

## 10. 제외 범위
회원가입, 로그인, 결제, 진도 저장, 관리자 CMS, 댓글, 수료증, AI 챗봇, 동영상 강의 플레이어는 제외한다.

## 11. 성공 기준
| 기준 | 목표 |
|---|---|
| 탐색성 | 원하는 세부교육내용까지 3클릭 이내 도달 |
| 콘텐츠 완성도 | 20개 강의가 모두 표시 |
| 반응형 | 375px 모바일 화면에서도 가독성 확보 |
| 사용성 | 현재 선택된 강의와 목차가 명확히 보임 |
| 확장성 | 데이터와 화면 코드가 분리되어 향후 CMS 전환 가능 |
