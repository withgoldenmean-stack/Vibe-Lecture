# 3. AX/DX 과제란 무엇인가?

## 3.1 AX/DX 과제의 쉬운 정의

AX/DX 과제란 다음과 같이 정의할 수 있다.

사람이 반복적으로 처리하던 업무 중 일부 또는 전체를 시스템, AI, API, 데이터 처리 기능을 활용해 더 빠르고 정확하게 수행하도록 바꾸는 업무 단위

쉽게 말하면 AX/DX 과제는 다음과 같은 업무이다.

- 매번 같은 양식으로 작성하는 업무

- 매번 비슷한 문구를 만드는 업무

- 매번 엑셀을 정리하는 업무

- 매번 이메일이나 문자를 보내는 업무

- 매번 보고서를 만드는 업무

- 매번 SNS에 게시하는 업무

- 매번 승인받고 기록해야 하는 업무

- 매번 데이터를 보고 판단해야 하는 업무

## 3.2 AX/DX 과제의 예시

| 업무            | 기존 방식                   | 자동화 방식                         |
|-----------------|-----------------------------|-------------------------------------|
| SNS 홍보글 작성 | 담당자가 매번 직접 작성     | AI가 초안 생성                      |
| 수강신청자 관리 | 엑셀에 수동 입력            | 신청 폼 → DB 자동 저장              |
| 보고서 작성     | 자료 취합 후 수동 작성      | 데이터 업로드 → AI 보고서 초안 생성 |
| 게시물 등록     | Facebook에 직접 접속해 게시 | 승인 후 API 자동 게시               |
| 고객 문의 분류  | 사람이 읽고 분류            | AI가 문의 유형 자동 분류            |
| 리뷰 분석       | 담당자가 리뷰를 읽고 정리   | AI가 긍정·부정·개선점 분석          |
| 예약 알림       | 담당자가 문자 발송          | 예약시간에 자동 알림 발송           |

# 4. 자동화에 적합한 업무와 부적합한 업무

## 4.1 자동화에 적합한 업무

자동화에 적합한 업무는 보통 다음 특징을 가진다.

| 기준            | 설명                             | 예시                        |
|-----------------|----------------------------------|-----------------------------|
| 반복성          | 자주 반복된다                    | 매주 SNS 게시글 작성        |
| 규칙성          | 처리 방식이 일정하다             | 신청자 명단 정리            |
| 데이터화 가능성 | 입력·출력 정보가 명확하다        | 수강생 이름, 과정명, 신청일 |
| 시간소모        | 사람이 처리하는 시간이 길다      | 월간 보고서 작성            |
| 오류위험        | 수작업 실수가 자주 발생한다      | 예약 게시 누락              |
| 효과성          | 자동화하면 비용·시간 절감이 크다 | 문의 응대 자동화            |
| 추적 필요성     | 누가 처리했는지 기록이 필요하다  | 승인 이력 관리              |

## 4.2 자동화에 부적합하거나 주의가 필요한 업무

모든 업무를 자동화하면 안 된다.  
특히 다음 업무는 사람이 최종 판단해야 한다.

| 업무 유형            | 주의 이유        | 처리 방향            |
|----------------------|------------------|----------------------|
| 최종 의사결정        | 책임 소재가 큼   | AI는 보조, 사람 승인 |
| 민감한 개인정보 처리 | 법적 위험        | 최소 수집, 마스킹    |
| 법률·세무 판단       | 전문 책임 필요   | 전문가 검토 필요     |
| 채용·평가·징계       | 차별·공정성 위험 | 자동화 제한          |
| 외부 공개 콘텐츠     | 브랜드·법적 위험 | 게시 전 승인 필수    |
| 고객 클레임 대응     | 감정적 맥락 중요 | AI 초안 + 사람 검토  |
| 재무·계약 결정       | 손실 위험        | AI 분석 + 사람 결정  |

# 5. 업무를 5가지 유형으로 나누기

AX/DX 과제를 발굴할 때 가장 중요한 첫 단계는 업무를 유형별로 나누는 것이다.

## 5.1 반복업무

### 의미

매번 같은 방식으로 반복되는 업무이다.

### 예시

- 매주 게시물 올리기  

- 매월 보고서 만들기  

- 신청자 명단 정리  

- 안내 문자 발송  

- 파일명 정리  

- 이메일 발송  

- 예약 확인

### 자동화 방식

시스템 자동화, 예약 실행, 템플릿 처리에 적합하다.

### 비개발자 설명

“내가 매번 똑같이 하는 일이라면 자동화 가능성이 높습니다.”

## 5.2 생성업무

### 의미

글, 문서, 이미지, 보고서, 이메일처럼 무언가를 새로 만들어야 하는 업무이다.

### 예시

- 홍보문구 작성  

- 블로그 글 작성  

- 보고서 초안 작성  

- 이메일 답변 작성  

- 카드뉴스 문구 작성  

- 강의안 작성  

- 사업계획서 초안 작성

### 자동화 방식

생성형 AI 활용에 적합하다.

### 비개발자 설명

“매번 글을 새로 써야 하는 일은 생성형 AI가 도와줄 수 있습니다.”

## 5.3 분석업무

### 의미

데이터나 문서를 보고 의미를 찾아야 하는 업무이다.

### 예시

- 매출 데이터 분석  

- 고객 리뷰 분석  

- 설문 결과 요약  

- 경쟁사 비교  

- 상권 데이터 분석  

- 고객 문의 유형 분류  

- 게시물 성과 비교

### 자동화 방식

AI 분석, Python 분석, 대시보드, 리포트 자동화에 적합하다.

### 비개발자 설명

“자료를 보고 ’무슨 뜻이지?’를 해석해야 하는 일은 AI 분석 후보입니다.”

## 5.4 판단업무

### 의미

최종 선택과 책임이 필요한 업무이다.

### 예시

- 가격을 올릴지 결정  

- 광고비를 늘릴지 결정  

- 게시물을 최종 승인  

- 고객 클레임 대응 방향 결정  

- 신제품 출시 결정  

- 지원사업 신청 여부 결정

### 자동화 방식

완전 자동화보다 AI 보조가 적합하다.

### 비개발자 설명

“결정의 책임이 큰 일은 AI가 대신 결정하는 것이 아니라, 판단을 도와주는 방식으로 설계해야 합니다.”

## 5.5 민감업무

### 의미

개인정보, 법률, 재무, 저작권, 평판 리스크가 큰 업무이다.

### 예시

- 고객 개인정보 처리  

- 수강생 얼굴 사진 활용  

- 계약서 검토  

- 결제정보 처리  

- 세무 판단  

- 법률 문서 작성  

- 외부 공개 광고 작성

### 자동화 방식

제한적 자동화, 검수 필수, 보안 설계 필요.

### 비개발자 설명

“실수했을 때 법적 문제가 생기는 업무는 자동화보다 통제와 승인 설계가 먼저입니다.”

# 6. AX/DX 과제 발굴 5단계

## 6.1 1단계: 업무 목록화

먼저 내가 하는 업무를 최대한 많이 적는다.

### 질문

- 하루 중 반복적으로 하는 업무는 무엇인가?

- 매주 반드시 하는 업무는 무엇인가?

- 매월 보고하거나 정리하는 업무는 무엇인가?

- 매번 새로 작성하는 문서나 콘텐츠는 무엇인가?

- 누군가에게 확인받아야 하는 업무는 무엇인가?

- 실수하면 문제가 커지는 업무는 무엇인가?

### 예시

패션학원 운영자의 업무 목록:

| 업무명               | 설명                            |
|----------------------|---------------------------------|
| 수강생 모집글 작성   | Facebook, Instagram 홍보글 작성 |
| 과정 정보 정리       | 개강일, 수강료, 대상 정리       |
| 수강문의 응대        | 전화·메시지 문의 답변           |
| 수강신청자 명단 관리 | 엑셀로 신청자 정리              |
| 수업사진 관리        | 작품사진, 수업사진 저장         |
| 게시물 등록          | Facebook Page에 게시            |
| 게시 성과 확인       | 좋아요, 댓글, 문의 수 확인      |

## 6.2 2단계: 업무 유형 분류

각 업무를 반복, 생성, 분석, 판단, 민감업무로 나눈다.

| 업무명               | 반복 | 생성 | 분석 | 판단 | 민감 |
|----------------------|------|------|------|------|------|
| 홍보문구 작성        | 중간 | 높음 | 낮음 | 중간 | 중간 |
| 수강신청자 명단 정리 | 높음 | 낮음 | 중간 | 낮음 | 높음 |
| 게시물 승인          | 낮음 | 낮음 | 중간 | 높음 | 중간 |
| 리뷰 분석            | 중간 | 낮음 | 높음 | 중간 | 낮음 |
| 수강생 사진 게시     | 중간 | 낮음 | 낮음 | 높음 | 높음 |

## 6.3 3단계: 문제 강도 파악

자동화할 업무는 단순히 “귀찮은 일”이 아니라 “문제가 큰 일”이어야 한다.

### 문제 강도 질문

1.      이 업무에 시간이 많이 걸리는가?  

2.    담당자가 바뀌면 품질이 달라지는가?  

3.    실수가 자주 발생하는가?  

4.    실수했을 때 고객 불만이나 손실이 생기는가?  

5.    업무 결과가 기록으로 남아야 하는가?  

6.    자동화하면 매출, 비용, 품질, 속도에 도움이 되는가?

## 6.4 4단계: 자동화 가능성 판단

다음 기준으로 자동화 가능성을 판단한다.

| 기준        | 질문                         |
|-------------|------------------------------|
| 입력 명확성 | 어떤 정보를 넣어야 하는가?   |
| 출력 명확성 | 결과로 무엇이 나와야 하는가? |
| 규칙성      | 처리 방식이 일정한가?        |
| 데이터 존재 | 필요한 데이터가 있는가?      |
| 예외 적음   | 예외상황이 너무 많지 않은가? |
| 승인 가능   | 사람이 검토할 단계가 있는가? |
| API 가능    | 외부 서비스와 연결 가능한가? |

## 6.5 5단계: 우선순위 선정

AX/DX 후보가 여러 개라면 우선순위를 정해야 한다.

### 추천 기준

| 기준                      | 점수  |
|---------------------------|-------|
| 반복성이 높은가?          | 1~5점 |
| 시간이 많이 걸리는가?     | 1~5점 |
| 자동화 효과가 큰가?       | 1~5점 |
| 구현 난이도가 낮은가?     | 1~5점 |
| 리스크가 낮은가?          | 1~5점 |
| 데이터가 준비되어 있는가? | 1~5점 |

총점이 높은 업무부터 MVP 후보로 선정한다.

# 7. AX/DX 후보 평가표

교육생에게 다음 표를 작성하도록 한다.

| 후보 업무            | 반복성 | 시간소모 | 효과성 | 구현난이도 | 리스크 | 데이터준비 | 총점 |
|----------------------|--------|----------|--------|------------|--------|------------|------|
| 홍보문구 생성        | 5      | 4        | 5      | 4          | 3      | 5          | 26   |
| Facebook 예약 게시   | 4      | 3        | 5      | 3          | 3      | 4          | 22   |
| 수강생 개인정보 관리 | 4      | 4        | 4      | 2          | 1      | 3          | 18   |
| 수강생 사진 자동게시 | 3      | 3        | 3      | 2          | 1      | 2          | 14   |

### 해석 예시

- 홍보문구 생성은 반복성, 효과성, 데이터 준비도가 높아 자동화 우선순위가 높다.

- 수강생 사진 자동게시는 초상권과 저작권 리스크가 높아 MVP에서는 제외하거나 승인 절차를 강화해야 한다.

- 개인정보 관리는 중요하지만 보안 리스크가 높으므로 초기 자동화 대상보다는 별도 보안 설계가 필요하다.

# 8. AX/DX 과제 발굴 사례

## 8.1 사례 1: 패션학원 마케팅 자동화

### 현재 문제

패션학원 원장은 매번 개강 과정마다 Facebook 홍보문구를 직접 작성한다.  
이미지를 고르고, 직원에게 확인하고, 다시 수정하고, Facebook에 직접 올린다.

### AX/DX 후보

| 업무           | 자동화 가능성 | 설명                           |
|----------------|---------------|--------------------------------|
| 과정 정보 등록 | 높음          | 정해진 입력폼으로 가능         |
| 홍보문구 생성  | 높음          | AI가 초안 작성 가능            |
| 이미지 선택    | 중간          | 보관함에서 선택 가능           |
| 원장 승인      | 중간          | 사람 승인 필요                 |
| Facebook 게시  | 높음          | API 연동 가능                  |
| 게시 실패 대응 | 높음          | 재시도 큐 가능                 |
| 성과분석       | 중간          | API 제공 지표 범위에 따라 가능 |

### MVP 추천 과제

과정 정보 기반 AI 홍보문구 생성 → 승인 후 Facebook 예약 게시

## 8.2 사례 2: 컨설팅 보고서 자동화

### 현재 문제

컨설턴트가 현장방문 후 메모, 사진, 매출자료를 정리해 보고서를 작성한다. 시간이 오래 걸리고 보고서 품질이 담당자마다 다르다.

### AX/DX 후보

| 업무               | 자동화 가능성 | 설명             |
|--------------------|---------------|------------------|
| 현장진단 항목 입력 | 높음          | 입력폼으로 가능  |
| 사진 업로드        | 높음          | 파일 저장 가능   |
| 매출자료 분석      | 중간          | 데이터 형식 필요 |
| 보고서 초안 생성   | 높음          | AI 생성 가능     |
| 최종 컨설팅 의견   | 낮음          | 전문가 판단 필요 |
| PDF 출력           | 높음          | 자동화 가능      |

### MVP 추천 과제

현장진단 입력 → AI 보고서 초안 생성 → 전문가 검토 → PDF 출력

## 8.3 사례 3: 공공기관 지원사업 관리

### 현재 문제

공공기관 담당자가 신청기업 자료를 엑셀로 관리하고, 평가결과를 수동 취합하며, 선정 안내문을 개별 발송한다.

### AX/DX 후보

| 업무           | 자동화 가능성 | 설명                |
|----------------|---------------|---------------------|
| 신청 접수      | 높음          | 온라인 폼           |
| 서류 누락 확인 | 높음          | 체크리스트          |
| 평가표 취합    | 높음          | DB 저장             |
| 선정 결과 통보 | 높음          | 자동 알림           |
| 민원 대응      | 중간          | AI 초안 + 사람 확인 |
| 최종 선정 판단 | 낮음          | 사람 책임 필요      |

### MVP 추천 과제

신청 접수 → 서류 누락 체크 → 평가표 취합 → 선정결과 알림

# 9. 비개발자가 AX/DX 과제 발굴 시 자주 하는 실수

## 9.1 너무 큰 과제를 고른다

나쁜 예:

“우리 회사의 모든 업무를 자동화하고 싶습니다.”

좋은 예:

“신규 수강생 모집 콘텐츠를 생성하고 승인 후 Facebook에 예약 게시하는 업무부터 자동화하겠습니다.”

## 9.2 사람이 판단해야 할 업무까지 자동화하려고 한다

나쁜 예:

“AI가 고객 클레임 답변을 자동으로 게시하게 하겠습니다.”

좋은 예:

“AI가 고객 클레임 답변 초안을 만들고, 담당자가 확인 후 발송하게 하겠습니다.”

## 9.3 데이터가 없는 업무를 자동화하려고 한다

나쁜 예:

“AI가 우리 매출을 분석해서 개선안을 내게 하겠습니다.”  
그런데 매출 데이터가 정리되어 있지 않음

좋은 예:

“먼저 매출 데이터를 월별, 상품별로 입력할 수 있는 구조를 만들고, 이후 AI 분석을 적용하겠습니다.”

## 9.4 예외상황을 고려하지 않는다

나쁜 예:

“예약 시간에 게시되면 됩니다.”

좋은 예:

“예약 게시가 실패하면 실패 사유를 저장하고, 3회 재시도하며, 최종 실패 시 관리자에게 알림을 보냅니다.”

## 9.5 유료화 가치를 고려하지 않는다

자동화 기능이 재미있어 보여도 고객이 돈을 내지 않으면 서비스가 되기 어렵다.

### 유료화 가능성이 높은 업무

- 자주 반복된다.

- 직접 하면 시간이 오래 걸린다.

- 실수하면 손해가 크다.

- 매출과 연결된다.

- 외부 고객에게 보여지는 결과물이다.

- 관리자나 원장이 성과를 확인하고 싶어 한다.

# 10. AX/DX 과제 선정 기준: 돈을 낼 만한가?

비개발자 교육에서 반드시 강조해야 할 질문이다.

이 기능이 있으면 고객이 실제로 돈을 낼까?

## 10.1 돈을 낼 가능성이 높은 자동화

| AX/DX 과제         | 돈을 낼 가능성이 높은 이유 |
|--------------------|----------------------------|
| 마케팅 콘텐츠 생성 | 시간 절감과 매출 연결      |
| 예약 게시          | 운영 부담 감소             |
| 게시 실패 알림     | 실무 리스크 감소           |
| 수강문의 자동 분류 | 상담 효율 개선             |
| 보고서 자동 생성   | 업무시간 절감              |
| 성과 리포트        | 의사결정에 도움            |
| 승인 워크플로우    | 조직 운영 안정화           |

## 10.2 돈을 내기 어려운 자동화

| AX/DX 과제               | 이유                 |
|--------------------------|----------------------|
| 단순 메모장              | 무료 도구가 많음     |
| 단순 파일 보관           | 차별성 부족          |
| 단순 To-do 리스트        | 대체재 많음          |
| 과도하게 복잡한 대시보드 | 소규모 고객에게 부담 |
| 재미용 AI 기능           | 업무성과와 연결 약함 |
