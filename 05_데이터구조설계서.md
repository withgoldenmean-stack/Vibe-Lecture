# 데이터구조 설계서

## 1. 문서 목적
본 문서는 AX/DX 20시간 과정 웹 서비스에서 사용할 강의 데이터 구조를 정의한다.

## 2. 데이터 관리 방식
MVP에서는 데이터베이스를 사용하지 않고 정적 데이터 파일로 관리한다.

권장 파일: `src/data/lessons.ts` 또는 `src/data/lessons.json`

## 3. TypeScript 타입 정의
```ts
export type LessonSection = {
  sectionId: string;
  title: string;
  content: string;
  activity?: string;
  checklist?: string[];
};

export type Lesson = {
  lessonId: string;
  lessonNo: number;
  unit: string;
  title: string;
  duration: string;
  target: string;
  output: string;
  summary: string;
  sections: LessonSection[];
};
```

## 4. 데이터 예시
```ts
export const lessons: Lesson[] = [
  {
    lessonId: "L001",
    lessonNo: 1,
    unit: "입문",
    title: "AX/DX와 바이브코딩 이해",
    duration: "60분",
    target: "비개발자 AX/DX 관심자",
    output: "개념 이해 체크리스트",
    summary: "DX, AX, 바이브코딩의 개념을 이해한다.",
    sections: [
      {
        sectionId: "S001-01",
        title: "학습 목표",
        content: "DX와 AX의 차이를 이해하고 바이브코딩이 비개발자에게 어떤 의미를 갖는지 설명한다.",
        activity: "내 업무에서 자동화가 필요한 장면 1개를 적는다.",
        checklist: ["DX와 AX의 차이를 설명할 수 있다.", "바이브코딩의 장점을 말할 수 있다."]
      }
    ]
  }
];
```

## 5. 필드 정의
| 필드 | 타입 | 설명 |
|---|---|---|
| lessonId | string | 강의 고유 ID |
| lessonNo | number | 교시 번호 |
| unit | string | 단원명 |
| title | string | 강의명 |
| duration | string | 강의 시간 |
| target | string | 대상 수강생 |
| output | string | 차시별 산출물 |
| summary | string | 강의 요약 |
| sections | array | 세부목차 목록 |

## 6. ID 규칙
| 항목 | 규칙 |
|---|---|
| lessonId | L001, L002, L003 |
| sectionId | S001-01, S001-02 |
| lessonNo | 1~20 |
| unit | 입문, AX/DX 기획, 개발문서 설계, AI·API·리스크, 운영·출시 |

## 7. 2차 DB 전환 시 테이블 구조
### lessons
| 필드 | 타입 |
|---|---|
| id | uuid |
| lesson_no | integer |
| unit | text |
| title | text |
| duration | text |
| target | text |
| output | text |
| summary | text |
| sort_order | integer |

### lesson_sections
| 필드 | 타입 |
|---|---|
| id | uuid |
| lesson_id | uuid |
| title | text |
| content | text |
| activity | text |
| checklist | jsonb |
| sort_order | integer |

## 8. 데이터 검수 기준
- 20개 강의가 모두 등록되어야 한다.
- 각 강의에는 최소 3개 이상의 세부목차가 있어야 한다.
- 각 강의에는 산출물이 반드시 있어야 한다.
- 각 세부목차에는 본문 content가 있어야 한다.
- 빈 문자열 데이터는 EmptyState로 처리한다.
