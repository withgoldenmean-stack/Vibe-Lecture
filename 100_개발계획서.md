# VIBELECTURE 개발계획서

## 1. 개발 목표
본 개발의 목표는 현재 폴더에 정리된 기획·설계 문서를 기준으로 「바이브코딩 활용한 AX/DX 자동화 기본과 실무 과정」 20시간 강의 콘텐츠를 탐색할 수 있는 반응형 웹 서비스를 완성하는 것이다.

개발 결과물은 다음 조건을 만족해야 한다.

- 20개 강의가 단원, 교시, 세부목차 단위로 탐색된다.
- 사용자는 Top Menu에서 단원 또는 강의를 선택할 수 있다.
- 사용자는 Left Menu에서 세부목차를 선택할 수 있다.
- Body 영역에는 선택된 세부목차의 상세 교육내용, 실습 활동, 산출물이 표시된다.
- PC, 태블릿, 모바일에서 모두 읽기 쉽고 조작하기 쉬워야 한다.
- 데이터와 화면 컴포넌트는 분리되어 향후 CMS, 검색, 진도관리, GPT API 기능으로 확장 가능해야 한다.

## 6. 환경변수 계획
GPT API 키는 소스코드에 직접 넣지 않고 환경변수로 관리한다.

### 6.1 생성할 파일
| 파일 | 목적 | 커밋 여부 |
|---|---|---|
| `.env.example` | 필요한 환경변수 이름과 예시를 공유 | 가능 |
| `.env.local` | 실제 사용자 API 키 입력 | 제외 권장 |
| `.gitignore` | `.env.local` 등 민감 파일 제외 | 가능 |

### 6.2 환경변수 항목
```env
# OpenAI API
OPENAI_API_KEY=

# Optional model setting
OPENAI_MODEL=gpt-4.1-mini

# App
NEXT_PUBLIC_APP_NAME=AX/DX Vibe Coding Course
```

### 6.3 적용 원칙
- 프론트엔드 컴포넌트에서 `OPENAI_API_KEY`를 직접 읽지 않는다.
- GPT API 호출은 반드시 `src/app/api/ai/route.ts` 같은 서버 Route Handler에서 처리한다.
- `NEXT_PUBLIC_` 접두사가 붙은 값만 브라우저에 노출한다.
- `.env.local`은 실제 키 입력용이며 공유하지 않는다.
- MVP에서는 API 키 입력 구조만 만들고, 실제 AI 기능은 기본 화면 흐름을 해치지 않는 범위에서 샘플 API 상태 확인 정도로 제한한다.

## 9. 주요 기능 상세 계획

### 9.1 단원 선택
- 단원은 `입문`, `AX/DX 기획`, `개발문서 설계`, `AI·API·리스크`, `운영·출시`로 구성한다.
- 단원 선택 시 해당 단원의 첫 번째 강의를 자동 선택한다.
- Desktop에서는 탭 형태를 우선 적용한다.
- Mobile에서는 select 또는 compact menu를 적용한다.

### 9.2 강의 선택
- 강의는 교시 번호, 강의명, 산출물을 함께 식별할 수 있게 표시한다.
- 선택된 강의는 색상, 굵기, 경계선 등 복수 신호로 강조한다.

### 9.3 세부목차 선택
- Left Menu에 선택된 강의의 `sections` 목록을 표시한다.
- 세부목차 클릭 시 Body를 즉시 갱신한다.
- 현재 선택 세부목차에는 `aria-current`를 적용한다.

### 9.4 본문 출력
- 단원명
- 교시 번호
- 강의명
- 세부목차 제목
- 상세 교육내용
- 실습 활동
- 체크리스트
- 산출물
- 이전/다음 버튼

### 9.5 빈 상태
아래 상황을 반드시 처리한다.

| 상황 | 메시지 |
|---|---|
| 강의 없음 | 등록된 강의가 없습니다. |
| 세부목차 없음 | 등록된 세부목차가 없습니다. |
| 본문 없음 | 등록된 상세교육내용이 없습니다. |

## 10. 데이터 작성 기준
각 강의는 다음 필드를 가진다.

```ts
export type Lesson = {
  lessonId: string;
  lessonNo: number;
  unit: string;
  title: string;
  duration: string;
  target: string;
  output: string;
  summary: string;
  sections: LessonSection[];
};
```

각 세부목차는 다음 필드를 가진다.

```ts
export type LessonSection = {
  sectionId: string;
```
