# 반응형·상태·라우팅 설계서

## 1. 문서 목적
본 문서는 웹 서비스의 반응형 화면 전환, 상태 관리, URL 라우팅 방식을 정의한다.

## 2. 반응형 기준
| 구분 | 화면 너비 | UI |
|---|---:|---|
| Mobile | 0~767px | 1열 구조 |
| Tablet | 768~1023px | 접힘형 Left Menu |
| Desktop | 1024px 이상 | 2열 구조 |

## 3. Desktop 레이아웃
```text
Top Menu
↓
Unit Tabs
↓
Lesson Selector
↓
Left Menu + Body
```

Left Menu는 좌측 280px 고정, Body는 나머지 영역을 사용한다.

## 4. Mobile 레이아웃
```text
Top Menu
↓
단원 선택 드롭다운
↓
강의 선택 드롭다운
↓
세부목차 선택 드롭다운
↓
Body
```

모바일에서는 Left Menu를 항상 펼쳐두지 않는다.

## 5. 상태 관리
### 기본 상태
```ts
selectedUnit
selectedLessonId
selectedSectionId
searchKeyword
```

### 상태 변경 규칙
| 액션 | 변경 상태 |
|---|---|
| 단원 선택 | selectedUnit 변경, 첫 강의 선택 |
| 강의 선택 | selectedLessonId 변경, 첫 세부목차 선택 |
| 세부목차 선택 | selectedSectionId 변경 |
| 검색어 입력 | searchKeyword 변경 |

## 6. URL 라우팅 방식
MVP에서는 단일 페이지 방식으로 구현한다.

```text
/
```

선택 상태는 쿼리 파라미터로 관리할 수 있다.

```text
/?lesson=L008&section=S008-02
```

## 7. 향후 라우팅 확장
```text
/
/lessons
/lessons/[lessonId]
/lessons/[lessonId]/[sectionId]
/search?q=keyword
```

## 8. 접근성
- 키보드로 메뉴 이동이 가능해야 한다.
- 현재 선택 메뉴에는 aria-current를 적용한다.
- 드롭다운에는 label을 제공한다.
- 색상만으로 선택 상태를 구분하지 않는다.

## 9. 완료 기준
- PC에서 2열 레이아웃이 정상 표시된다.
- 모바일에서 1열 레이아웃이 정상 표시된다.
- 강의 선택 시 세부목차와 Body가 동기화된다.
- URL로 특정 강의와 세부목차에 접근할 수 있다.
